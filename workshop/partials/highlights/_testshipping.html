Send message to shipping

Expose the shipping service, make sure to provide the namespace_name: 

kubectl port-forward svc/shipping 8000:80 -n namespace_name

Open another terminal and run the following command which sends a test message to the shipping microservice running as a pod in your local Kubernetes cluster.

curl -X POST \
http://localhost:8000/shipping \
-H 'Content-Type: application/json' \
-H 'cache-control: no-cache' \
-d '{ 
  "id": "123-45-8876",
  "name": "shipment 500" 
}'

Look at the terminal windows and observe messages being created in the shipping and consumed in the stream. Notice the extra message that you added to the shipping service. You should also look at the OCI console. Refresh the table and watch your message being published to the stream.
