What is Cloud Native?

Cloud native technologies are characterized by the use of containers, microservices, serverless functions, development pipelines, and infrastructure expressed as code. Cloud native enables faster software development and the ability to build applications that are resilient, manageable, observable, and dynamically scalable to global enterprise levels.

What is Cloud Native

  • anything that uses a cloud provider's api
  • APIs on top of resources eliminates middle men and human delay.
  • APIs can be invoked from an application - building applications that call these APIs to create resources - provision environments - are are the next logical step in infrastructure management.
  • Now that infrastructure can be orchestrated using application code, coding and SDLC principles can be applied to infra management - Infrastructure as code.
  • Moving up from infrastructure management to application development, we now can build applications that can leverage these mechanisms.
  • an application that can create more compute capacity when it requires it. (although this is better handled outside an application in many ways.)
